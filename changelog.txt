v0.3.1c | 2026-05-11
* Renamed the farming inspector panel to **Farmland Preparation** and simplified it to a single **Farmland Preparation Automation** checkbox.
* Removed detailed farming preparation analysis from the inspector panel and removed the **atd_set_farming_analysis_debug** toggle.
* Reduced farming automation tick overhead by keeping routine status updates to compact summaries; detailed per-origin farming diagnostics remain available through **atd_farming_dump_all_towers**.
* Added **atd_set_re_enable_farming_on_load** / **atd_re_enable_farming_on_load** and the matching **ATDsettings.json** key **reEnableFarmingOnLoad** (default: true). On load, ATD re-enables farming automation for towers whose managed designations appear to be farmland preparation work.
* Replaced the old experimental filling sequencing with a vehicle clear-out guard. Before final filling commits, ATD orders vehicles in the fill area to park at the tower, waits for moving vehicles to leave, rechecks before committing, and proceeds if a vehicle appears stuck or paused.
* Delays switching the tower to farmable dump products until immediately before final fill designations are committed, preventing dirt/topsoil from being used on remaining preparation work.
* Temporarily releases empty tower-assigned trucks during final filling so they can help with dirt deliveries, while keeping at least one truck assigned to the tower for alerts and leftover debris work. Loaded tower trucks stay assigned until they dump their final prep cargo, and released trucks are reassigned to the tower when filling is done.
* Kept completed temporary preparation designations active until final filling commits, so fresh spills on prepared ground can still be scooped up.
* Added temporary sloped dumping support shoulders on exposed sides when adjacent terrain is below the target-1 preparation height, reducing dirt spill-off during final filling near steep drops or ocean edges without cascading farming preparation outward.
* Reduced farming access/pathability check cost on large jobs by indexing reachable target tiles instead of checking every active designation for every visited path tile.
* Added a green one-time completion notification when farmland preparation and filling are all done.
* Added a green one-time notification when any vehicle depot completes an excavator. Toggle with **excavatorCompletionNotifications** in **ATDsettings.json** or **atd_set_excavator_completion_notifications**.

v0.3.1b | 2026-05-10
* Continued alpha testing for experimental farming designations.
* Reduced farming stutter by throttling expensive access/pathability checks.
* Added automatic farming automation startup for existing towers after load/initialization.
* Improved farming phase handling: suppresses ready-for-filling origins during preparation, keeps preparation holds active until tower-level filling, and keeps filling origins tracked until the whole tower is complete.
* Added automation-owned access ramp cleanup and ramp tile reservation so generated access ramps do not overwrite tracked farming origins.
* Added world lifecycle cleanup and save hooks: runtime farming scaffolding is restored to vanilla leveling orders before saves, re-analyzed after save completion, and cleared when a game terminates.

v0.3.1a | 2026-05-09
* Improved **ATDsettings.json** migration so generated defaults from older releases are upgraded to the current defaults while user-changed values are preserved.
* Recognizes both pre-0.3.1 ore purity defaults and 0.3.1-generated purity defaults as generated values during future migrations, so users who only changed unrelated settings still receive updated purity tuning.


v0.3.1 | 2026-05-09
* Ramp generation warnings now shown as a yellow icon next to the **Create Designations** button (tooltip contains the message):
    - "Ramp generation failed — no valid path found."
    - "Ramp placed but did not reach the surface — excavators may not be able to excavate."
    - "Couldn't find a valid path from the tower to the generated ramp. Check for access problems."
* Increased generated ramp safety margin around nearby buildings by 1 tile. Landslide can still reach buildings if built on deep dirt/sand.
* Added a bottom-flattening pass to reduce rough/bumpy excavation floors; designations floor should now be noticeably flatter. Can be toggled on/off with **atd_set_bottom_flattening on/off**.
* Tuned default **Low** and **Med** ore purity thresholds to reduce the jump from **Off**; existing **ATDsettings.json** values are preserved. Added **atd_reset_to_defaults** to try built-in defaults in memory.
* Renamed the **Target product** label to **Scanning filter** to avoid confusion with mining priority.


v0.3.0 | 2026-05-08

v0.2.4 | 2026-05-04
* Settings file renamed from **settings.json** to **ATDsettings.json** to avoid conflicts with other mods
* Existing **settings.json** files are still read as a legacy fallback and migrated into the generated **ATDsettings.json** format
* **ATDsettings.json** is no longer distributed inside the mod ZIP — it is generated automatically in the mod folder on first run, populated with the current defaults and inline documentation
* Settings file now contains a **settingsVersion** stamp; when the mod version advances the file is automatically migrated (user values are preserved and any new keys are added)
* Added **atd_save_settings** console command to write the current in-memory global defaults back to **ATDsettings.json** at any time
* Added global defaults for whether the Terrain Designations and Ore Composition panels start expanded or collapsed, configurable via **ATDsettings.json** or console commands.
* Fixed Corridor clearance modifier-click handling so Shift/Ctrl jumps directly between Off and 2 instead of requiring multiple clicks.
